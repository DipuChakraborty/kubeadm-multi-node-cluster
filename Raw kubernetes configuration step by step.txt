Enable kernel modules:
A kernel module (or loadable kernel mode) is an object file that contains code that can extend the 
kernel functionality at runtime (it is loaded as needed); When a kernel module is no longer needed, 
it can be unloaded.
kernel modules is very important for Kubernetes networking and container runtime to work correctly.


Load kernel modules (temporary)
root@k8s-master:~# sudo modprobe overlay                   #It turns on OverlayFS, which lets your system "stack" folders into layers.It is required for Docker and Kubernetes to manage container images.It only stays active until a reboot unless added to a config file.
root@k8s-master:~# sudo modprobe br_netfilter              #This command enables bridge networking features in your Linux kernel.It is required for Kubernetes to allow traffic to flow correctly between containers.Like the overlay module, it will disappear after a reboot unless saved to a config file.

ensures the modules are automatically loaded after reboot
root@k8s-master:~# cat <<EOF | sudo tee /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF

Network settings:
This step is critical for Kubernetes networking to actually work
This step is Required for pod networking

To Allow bridge network traffic (used by containers) to pass through iptables
net.bridge.bridge-nf-call-iptables = 1
net.bridge.bridge-nf-call-ip6tables = 1

To allow Linux to forward packets between interfaces
net.ipv4.ip_forward = 1

root@k8s-master:~# cat <<EOF | sudo tee /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF
root@k8s-master:~# sudo sysctl --system

This command allows your Linux system to route network traffic between containers and the internet.
It specifically tells the firewall to handle bridge traffic and enables standard IP forwarding.
root@node1u:~# cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward = 1
EOF

This command activates all the network settings you just saved in those configuration files.
root@node1u:~# sudo sysctl --system


root@node2u:~# cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables = 1

STEP 3: Install Kubernetes (ALL NODES):
=======================================

Install required packages:
sudo apt update
sudo apt install -y apt-transport-https ca-certificates curl gpg

These are support tools:

apt-transport-https → allows APT to use HTTPS repos
ca-certificates → verifies secure connections
curl → download Kubernetes repo key
gpg → verify package authenticity

👉 Without these, Kubernetes repo setup will fail


root@k8s-master:~# sudo apt update
root@k8s-master:~# sudo apt install -y apt-transport-https ca-certificates curl gpg

Add Kubernetes repository (NEW METHOD):
Create keyring directory. This command creates a secure directory to store the public keys for your software repositories.
sudo mkdir -p /etc/apt/keyrings

Add GPG key
This downloads the official Kubernetes security key and converts it into a format Linux can read.
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.35/deb/Release.key \
| sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg

✅ Why this matters
Ensures packages are trusted and not tampered
Required by modern Ubuntu (no apt-key anymore)


Add repository
echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] \
https://pkgs.k8s.io/core:/stable:/v1.35/deb/ /' \
| sudo tee /etc/apt/sources.list.d/kubernetes.list


Install Kubernetes components
sudo apt update
sudo apt install -y kubelet kubeadm kubectl

What each component does
🔹 kubelet (MOST IMPORTANT)
Runs on every node
Talks to container runtime (containerd)
Actually runs Pods

👉 Think of it as:
“node agent”

🔹 kubeadm
Used to:
Initialize master (kubeadm init)
Join workers (kubeadm join)

👉 Think:
“cluster setup tool”

🔹 kubectl
CLI to control cluster

👉 Example:

kubectl get nodes

👉 Think:
“remote control for Kubernetes”



Hold package versions (VERY IMPORTANT):
sudo apt-mark hold kubelet kubeadm kubectl

Prevents:

Automatic upgrades
Version mismatch between nodes

👉 Kubernetes is VERY sensitive to version differences

Enable kubelet:
sudo systemctl enable --now kubelet

STEP 4: Initialize MASTER NODE:
================================

👉 Run ONLY on 192.168.40.131:

This command launches your cluster by setting up the master node.It uses 192.168.40.131 as the main control 
point for all communication.It reserves a private IP range (10.244.0.0/16) specifically for your pods.It 
creates the security certificates and the "join" code needed for worker nodes.
sudo kubeadm init \
--apiserver-advertise-address=192.168.40.131 \
--pod-network-cidr=10.244.0.0/16

👉 Explanation:

--apiserver-advertise-address: Master IP
--pod-network-cidr: required for CNI (Flannel/Calico)


Configure kubectl (MASTER only)
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
